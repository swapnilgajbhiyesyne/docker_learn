A Beautiful, Beginner‑Friendly Guide to docker run

Learn the essentials: Image tags, interactive mode (-it), port mapping (-p), volumes (-v), and how to inspect and read logs. Everything you need to get productive quickly.

1 · Running Containers with Tags

Choose the exact image version with tags

What it is: Each image version is labeled with a tag like :latest or :4.0. If you don’t specify one, Docker pulls :latest.

Why Locking versions avoids surprises when upstream images change.
# runs latest version
docker run redis
# runs Redis version 4.0
docker run redis:4.0

Tip: check tags on Docker Hub. Many projects also publish :alpine or :slim variants for smaller images.

2 · Interactive Containers

Use -it for keyboard input & prompts

  • -i keeps STDIN open (send input to the container)
  • -t allocates a pseudo‑TTY (nice prompts & colors)
docker run -it ubuntu

Without -it, interactive programs may look frozen because there’s no attached terminal.

3 · Port Mapping

Expose your app to the outside with -p

Containers run in their own network namespace. Use -p <host>:<container> so your host can reach the app.

docker run -p 80:5000 myapp
Why App listens on 5000 inside the container, but you visit http://localhost:80 on your machine.
4 · Persisting Data

Keep your data safe with -v volumes

Deleting a container also deletes its writable layer. Map a host folder to the container to persist data.

docker run -v /opt/data:/var/lib/mysql mysql

Use docker volume create for named volumes; they’re portable and managed by Docker.

5 · Inspecting Containers

See everything with docker inspect

docker inspect <container-id>

Outputs JSON: environment, mounts, network, state, entrypoint, and more—perfect for debugging.

6 · Viewing Logs

Read output from background apps with docker logs

docker logs <container-id>

Use with -f to follow logs live. Handy after running -d (detached mode).

Quick recap

  • Tags pin versions (redis:4.0)
  • -it for interactive terminals
  • -p exposes ports (host:container)
  • -v maps storage to persist data
  • inspect dumps full config; logs shows output

Pro tip: Combine flags! Example: docker run -d -p 8080:80 --name web nginx:alpine
